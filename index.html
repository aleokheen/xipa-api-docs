This is XIPA API documentation

With love only for Zhenyok

The base URL is:

https://api.xipa.ru

How to use

POST or GET or PUT or DELETE requests

Use HTTP method for CRUD metolodogy. If you`re creating something, use POST.

If you`re reading (getting) something, use GET

If you`re updating something, use PUT

If you`re deleting something, use DELETE

Rate limits

First you need to know that the server is rate limited. It means that every HTTP client can send limited requests per time unit. You can see rate limit info in response headers

Header name Description
X-RateLimit-Limit Amount of total requests that you can send
X-RateLimit-Remaining Amount of requests that you can send
Retry-After Timestamp when you can retry

Validation

Do data validation in client before send it to API

If client sends invalid data server will return a string with validation errors. Each one starts with new line
